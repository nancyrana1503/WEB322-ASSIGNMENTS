Project Description

  • Backend built using Express.js for handling server requests and routing.
  • Provides structured dataset including species, location, habitat, observation date, and verification status.
  • Implements API endpoints to:
    • Retrieve all wildlife sightings
    • Filter sightings by verification status
    • Get unique species list
    • Filter sightings by habitat type
    • Search sightings by animal name
    • Retrieve the three most recent sightings
  • Frontend uses HTML5, CSS, and responsive design for a clean user interface.
  • Data is presented in a clear, tabular format for easy analysis of trends and patterns.
  • Implements interactive features like search and filters to improve user experience.
  • Combines real-time data handling with a responsive UI for efficient monitoring of wildlife sightings.

API Endpoints

GET /api/sightings

Returns all wildlife sighting records
GET /api/sightings/verified

Returns only verified sightings
GET /api/sightings/species-list

Returns unique species names
GET /api/sightings/habitat/forest

Returns all sightings in forest habitat along with the count
GET /api/sightings/search/eagle

Searches for the first sighting of an eagle
GET /api/sightings/find-index/moose

Finds the index of the moose sighting
GET /api/sightings/recent

Returns the 3 most recent sightings
